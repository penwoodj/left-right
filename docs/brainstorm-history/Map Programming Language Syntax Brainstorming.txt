/**
 * Abstraction is compression.
 * abstraction can lose information or "cost" something in lost data.
 * There are zero cost abstractions and those are the holy grail.
 * Increasing the total number of abstractions can reduce the cost because each abstraction
 * contains less information that is being obscured but the utility of the abstractions is lost.
 * The more abstract the ideas the more useful they are in a sense because you can communicate
 * the same information with less abstractions, which in a sense is the whole point of abstraction.
 * Less abstractions that are zero costs tends to be a useful way of rank ordering models of thinking.
 * A model is a collection of abstractions we find useful in explaining the data.
 * However, the more abstract you are in the ways you can explain everything it can be in a
 * sense harder to learn so we scope the types of problems certain abstractions can solve
 * in order to maintain zero cost with less abstract ideas.
 * 
 * You can express any explanation/compression of information with varying number of abstractions.
 * 
 * Ways abstractions can vary:
 * - Loss of information 
 * - Total number of Abstractions
 * - Utility of an abstraction in specific concepts defined by ease of explanation and readability
 * - Scoping an abstraction to a specific context
 * - Thoroughness of the group of abstraction
 * - The amount of varying way the same abstractions can explain the same data
 * 
 * Each of these varying ways seems to be a sliding scale of pros and cons based on preference
 * and objective metrics of utility in varying contexts.
 * 
 * In human knowledge generally in academia, the fewest number of ideas that explain all 
 * of the data tend to be the preferred models.
 * 
 */

/**
 * APL Map Language Variant Rules:
 * Everything is evaluated left to right unless there is a scope operator present
 *
 * [] List Creation scope operator
 * - looks for next comma between [], and evaluates expression between starting [ or previous comma and that found comma
 * 
 * {} Map Creation scope operator
 * - looks for next comma between [], and evaluates expression between starting [ or previous comma and that found comma
 * - Evaluation of expression first splits each expression with : operator and evaluates the left expression and then the right expression
 *   - the left expression gets stored as the alias/key to the resulting value of the right expression (variable) for the 'remainder of the scope'(i.e. until closing bracket })
 *   - the evaluated keys and values available for access overall map after the expression is evaluated (e.g. { key: 1 } @ )
 *   - keys (i.e. value to left of :) can just be characters which will be assumed to be a string assuming no
 *   - values are evaluated as implicit expression scope (as if using expression scope operator)
 * 
 *   - if the last expression does not contain a `:` symbol between last comma and ending } that is the evaluated as an expression and is returned when the map is reference, thus making the Map Data Type into a Map Operator
 *   - if every Map Scope Sub Expression has a : operator in it then then Map Data type is returned
 * 
 *   - _<: left argument or whatever is passed into the left of the Map or Map Reference
 *     - Left is preferred to be data over operators
 *   - _>: right argument or whatever is passed into the right of the Map or Map Reference

 * `` String scope operator
 * - without unescaped {} in string `` just evaluates as a normal string
 * - for each {} in string the string becomes an operator
 * - `asdf: {_<}` `asdf: {x:_<@0, x+1}` becomes an unexecuted operator that takes a left argument 
 * - `asdf: {otherVar + 1}` is evaluated as a string not operator as the expression doesn't have an argument
 * - escaping works same as JS, and newlines must be explicit
 * 
 * - expressions are compositions of data and operators
 * 
 * () Expression scope operator
 * -
 * - Data types:
 *   - Map {}
 *   - List []
 *   - String ``
 *   - Number 0

 *   - Undefined undefined
 * - Operators are expression that get repeatedly evaluated contextually when the Map Operator is referenced as words or symbols

 * @ Get Operator
